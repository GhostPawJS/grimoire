# Spell Forge Demo

The GitHub Pages demo is a **static SPA** that runs entirely in the browser: **sql.js-fts5**
(WASM SQLite), an in-memory **virtual spell store** (SKILL.md content), and the same
metadata/event/health/note/registry code paths as the Node package. **Git is absent**
in the browser, so ranks stay at 0 and tiers read as **Uncheckpointed**.

## Build

```bash
npm run demo:build    # outputs to demo/
npm run demo:serve    # build + static server on http://localhost:4173
npm run demo:seed     # optional: refresh demo_registry_snapshot.json from AgentSkillHub
```

Design system lives in [`scripts/demo.css`](../scripts/demo.css) (system fonts only, no
external font requests). The HTML shell is generated by [`scripts/build_demo.mjs`](../scripts/build_demo.mjs).

## Search (no backend)

- **Local index**: pre-seeded `scout_index` rows (from `demo_registry_snapshot.json`, optionally refreshed via `demo:seed`). Search uses the real [`searchIndex`](../src/registry/search_index.ts) function against sql.js.
- **GitHub live**: browser `fetch` to `api.github.com` (CORS-enabled). Adoption uses the GitHub Contents API to download SKILL.md, then [`demoInscribe`](../src/demo/demo_operations.ts).

## Screens

| Screen        | Route        | Description |
|---------------|--------------|-------------|
| Dashboard     | `/`          | Stats, resonance leaderboard, recent events, health snapshot, quick actions |
| Chapters      | `/chapters`, `/chapters/:chapter` | Tabs per chapter, spell cards with resonance |
| Spell detail  | `/spell/:chapter/:name` | Body, frontmatter, validation, notes, events, provenance, edit/copy/delete/move/shelve |
| Inscribe      | `/inscribe`  | Create spell with live validation; supports prefill from Notes via sessionStorage |
| Notes         | `/notes`     | Drop notes, counts, route domain, distill into Inscribe |
| Catalogue     | `/catalogue` | Run [`demoCatalogue`](../src/demo/demo_operations.ts), view snapshot and heatmap |
| Scout         | `/scout`     | Local index search + GitHub live search and adopt |

## Honing / seal

The full **honing workshop** (seal actions, git-backed rank) is not available in the
browser demo because there is no git binary. Use the Node package for seal workflows.
